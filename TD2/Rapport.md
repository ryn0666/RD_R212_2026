# Rapport — TD2

## Points clés

### 1. CSS minimal et typographie
J’ai appris à créer un CSS minimal qui respecte le HTML existant. La largeur de lecture, l’interligne et la hiérarchie des titres rendent le contenu lisible sans layout complexe.

### 2. Flexbox pour le layout
J’ai utilisé Flexbox pour le menu de navigation et pour les trois articles de la section "Pourquoi". Cela permet aux éléments de s’adapter naturellement à la taille de l’écran et de rester lisibles en mobile.

### 3. Sélecteurs simples et robustes
L’usage de sélecteurs simples (nav, ul, article, blockquote) permet de styliser sans ajouter de classes inutiles. Le CSS reste maintenable et survit à un changement de contenu.

## Utilisation de l'IA

### Démarche
J’ai utilisé ChatGPT (modèle GPT-4 via interface web) pour m’aider à vérifier mes choix CSS et trouver des idées pour rendre le site responsive et harmonieux avec des tons beige.

### Apports et limites
L’IA m’a aidé à comprendre comment utiliser Flexbox pour que les articles se mettent automatiquement en colonne sur mobile, et pour proposer des couleurs harmonieuses. J’ai dû simplifier certaines suggestions pour que le CSS reste minimal et robuste.

### Prompts significatifs
1. "Peux-tu me proposer un CSS simple pour une page déjà structurée en HTML, responsive, avec Flexbox ?" → m’a aidé à structurer le layout et le responsive.
2. "Comment utiliser Flexbox pour que trois articles s’empilent automatiquement sur mobile ?" → a confirmé l’usage de `flex-wrap`.
3. "Propose une palette de couleurs beige cohérente pour un site minimaliste." → a permis de choisir un ton beige uniforme pour header et blockquotes.
